Segurança de Dados


Devido à necessidade de controle do acesso e compartilhamento das informações, a plataforma API Gateway habilita para cada cliente a criptografia dos dados enviados e recebidos através das integrações, considerando o fluxo e o workflow de cada cliente.

Caso a funcionalidade de criptografia esteja habilitada, serão aplicadas as seguintes regras:

  • Criptografia simétrica no padrão AES256;
  • Valor de Chave e Salt alinhados com cada cliente em tempo de planejamento de projeto;
  • Caso a feature de criptografia esteja ativa, o cliente irá utilizar as rotas criptografadas de cada funcionalidade para enviar os dados criprografados no seguinte modelo:

      {  
          “encrypted_content”: “@$RdSED#R34rFsdVD433rfds4”
      }

  • A rota criptografada deverá descriptografar os dados e desserializar o objeto para o domínio esperado dentro da rota principal para aplicar as regras de negócios necessárias;
  • Caso a feature de criptografia esteja ativa, o retorno de dados da API deverá ser criptografado para o seguinte modelo antes do retorno ser enviado para o cliente.
